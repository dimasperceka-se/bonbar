# Replit Prompt: Lapas Goods Request & Monitoring PWA

Copy and paste the following prompt into Replit:

---

Build a Progressive Web App (PWA) called **"Bon Barang Lapas"** for monitoring and requesting goods/supplies at a Correctional Facility (Lembaga Pemasyarakatan / Lapas) office. The app converts natural language text requests into a structured, pre-filled official request form.

## Tech Stack
- **Frontend:** React + Vite + TailwindCSS
- **Backend:** Node.js + Express
- **Database:** SQLite (with Prisma ORM) for easy Replit deployment
- **PWA:** Vite PWA plugin with offline support, installable on mobile/desktop
- **AI/NLP:** Use OpenAI API (or Anthropic Claude API) for parsing natural language requests
- **PDF Generation:** jsPDF + html2canvas for exporting filled forms
- **Auth:** Simple JWT-based authentication

## Core Feature Flow
1. User logs in (role-based: Requester, Approver/Kalapas, Admin)
2. User types a free-text request in natural language, e.g.:
   *"I need 2 reams of A4 paper, 1 bottle of black Epson L310 ink, and 5 rolls of clear tape for the Kamtib section"*
3. The system uses AI to parse the request and auto-fill the official **Bon Barang** form
4. User reviews, edits if needed, and submits
5. Request is stored, trackable, and can be approved/rejected by Kalapas
6. Approved requests can be exported as PDF matching the official format

## Form Structure (Bon Barang)
The output form must replicate this official format:

**Header (fixed):**
- KEMENTERIAN HUKUM DAN HAK ASASI MANUSIA
- KANTOR WILAYAH JAWA BARAT
- KEMENTERIAN IMIGRASI DAN PEMASYARAKATAN REPUBLIK INDONESIA
- DIREKTORAT JENDERAL PEMASYARAKATAN
- KANTOR WILAYAH JAWA BARAT
- LEMBAGA PEMASYARAKATAN KELAS IIA KUNINGAN
- Jl. Siliwangi No. 2 Kuningan 45513 Telp. (0232) 871136 Fax. (0232) 871130
- Laman: lapaskuningan.kemenkumham.go.id, Email: lp2kuningan@gmail.com
- Indonesian government logo (Garuda) placeholder
- Title: **BON BARANG**

**Table columns:**
| NO. | NAMA BARANG (Item Name) | BANYAKNYA (Quantity) | KETERANGAN (Notes/Purpose) |

**Footer:**
- Location and date (e.g., "Kuningan, 22 Desember 2025")
- Two signature blocks:
  - Left: "Mengetahui, Kalapas," with name
  - Right: "Ka Si Kamtib," (or requesting section head) with name
- Digital signature support (upload signature image or draw on canvas)

## AI Parsing Requirements
The NLP layer must extract:
- **Item name** (normalize common items: HVS paper, Kanebo, Sunlight, Superpel, Tinta Epson, Lakban, Spidol, Plastik, Binder Klip, etc.)
- **Quantity + Unit** (Rim, buah, pouch, botol, Roll, pak)
- **Purpose/Section** (e.g., "Seksi Kamtib", "Ruang Pendaftaran") → grouped into KETERANGAN column
- Support both Indonesian and English input
- Group items by purpose/section automatically

**Example parsing:**
Input: *"Need 1 ream of F4 paper, 2 reams A4, 1 Kanebo, 1 Sunlight pouch for Kamtib section, and 5 rolls of clear tape, 2 permanent markers for Registration room"*

Output JSON:
```json
{
  "items": [
    {"no": 1, "name": "Kertas HVS F4", "qty": "1 Rim", "purpose": "Untuk Keperluan Seksi Kamtib"},
    {"no": 2, "name": "Kertas HVS A4", "qty": "2 Rim", "purpose": "Untuk Keperluan Seksi Kamtib"},
    {"no": 3, "name": "Kanebo", "qty": "1 buah", "purpose": "Untuk Keperluan Seksi Kamtib"},
    {"no": 4, "name": "Sunlight", "qty": "1 pouch", "purpose": "Untuk Keperluan Seksi Kamtib"},
    {"no": 5, "name": "Lakban bening", "qty": "5 Roll", "purpose": "Untuk Keperluan Ruang Pendaftaran"},
    {"no": 6, "name": "Spidol Permanen", "qty": "2 buah", "purpose": "Untuk Keperluan Ruang Pendaftaran"}
  ]
}
```

## Pages / Screens
1. **Login** – Username/password with role selection
2. **Dashboard** – Stats: pending requests, approved this month, top requested items, low-stock alerts
3. **New Request (Main feature)**
   - Large text area: "Describe what you need..."
   - "Parse with AI" button
   - Preview of auto-filled form (editable table)
   - Add/remove/edit rows manually
   - Submit button
4. **My Requests** – List of submitted requests with status (Pending, Approved, Rejected, Fulfilled)
5. **Request Detail** – Full form view, approval history, export PDF
6. **Approval Queue** (Kalapas role) – Pending approvals with approve/reject actions
7. **Inventory Monitoring** – Item master list, stock levels, request frequency analytics
8. **Item Master** (Admin) – CRUD for standard items, units, and default sections
9. **Reports** – Monthly/quarterly request reports, exportable to Excel/PDF

## Database Schema
- **users** (id, username, password_hash, full_name, role, section, signature_url)
- **items** (id, name, default_unit, category, current_stock)
- **requests** (id, requester_id, status, created_at, approved_by, approved_at, location, request_date)
- **request_items** (id, request_id, item_name, quantity, unit, purpose, item_no)
- **approval_history** (id, request_id, action, actor_id, notes, timestamp)
